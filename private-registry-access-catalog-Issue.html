clarktlaugh commented on 30 Oct 2015

I would like to setup my ACL to allow anonymous access to the catalog, as well as pulling all images. Could someone help me with how to specify that?

Here's what I am currently using, but it isn't working:

acl:
  # Admin has full access to everything.
  - match: {account: "admin"}
    actions: ["*"]
  # User "public" can pull stuff.
  - match: {account: "public"}
    actions: ["pull"]
  # Trying to allow anonymous to access the catalog, pull stuff
  - match: {account: ""}
    actions: ["pull"]
  # Access is denied by default.

This is what I get when I try to hit https://my-registry.server.org/v2/_catalog in a web browser:

{"errors":[{"code":"UNAUTHORIZED",
"message":"access to the requested resource is not authorized",
"detail":[{"Type":"registry","Name":"catalog","Action":"*"}]}]}

kwk commented on 3 Nov 2015

Try to access your registry's catalog using curl curl -v https://YOUR_REGISTRY:5000/v2/_catalog and inspect the Www-Authenticate header in the response from the registry. It should give you a hint on what permissions are required to access the _catalog resource.

I've setup a docker-registry, authentication and LDAP server ready to use if you want to try out various scenarios: https://github.com/kwk/docker-registry-setup. At the bottom of the README I also write about manually querying the registry and then the auth server.

rojer commented on 5 Nov 2015

acl is the authorization part, i.e. when we know who are we talking to.
the part where we authenticate precedes that.
you didn't paste the users part of the config, but basically this line is crucial for allowing anonymous access:
"": {} # Allow anonymous (no "docker login") access.

this is from https://github.com/cesanta/docker_auth/blob/master/examples/reference.yml
